# 场景图生成 v1.1.0

把真实家具安全地放进参考场景。场景可以重做，产品身份不能重做；产品角度可以改变，产品刚性几何和全部细节不能改变。

## 1. 何时必须使用

当用户同时提供家具产品图与场景参考图，并要求以下任一任务时必须使用：

- 把产品放进参考场景；
- 参考另一张图的机位、角度、构图、软装或光影；
- 制作电商场景图、详情页场景图、社媒图或展厅效果图；
- 同一真实产品做多场景或批量视觉升级。

不用于：改变家具造型、结构、部件、颜色、材质或五金的新品设计；没有指定真实产品的纯室内灵感；店铺流量、投放、转化或销售数据分析。

## 2. 最高优先级

发生冲突时固定按以下顺序决策：

1. 产品事实与刚性几何；
2. 产品细节可核对；
3. 空间物理、机位和光影一致；
4. 卖点与渠道表达；
5. 氛围和审美。

产品保真拥有最终否决权。融合再自然，只要产品身份、几何或细节改变，就必须废稿。

## 3. 产品图与场景图严格分工

- 产品图是唯一产品事实源：决定轮廓、比例、部件数量、五金、连接工艺、缝线边型、材质、纹理、木纹、固有色、表面处理和 Logo。
- 场景参考图是拍摄条件源：决定房间、摆位、目标朝向、机位高度、地平线、消失点、焦段感、构图、软装和光影。
- 场景参考中的原家具只能用于理解位置和摄影关系，不能把它的分格、拉手、颜色、材质、腿型、底座或结构迁移到目标产品。
- 不能因为产品原图更容易生成，就默认沿用产品原图角度；也不能因为场景原家具占据某个四边形，就把目标产品硬拉进该四边形。

## 4. 素材门禁

生成前先在对话内建立锁定清单，不要求外部模板或脚本。

必须记录：

- 哪些图片是产品事实源，哪些是场景参考；
- 产品正面、背面、左右侧、顶面、底部可见区、内部与关键细节的覆盖情况；
- 可确认的整体比例、部件数量、五金数量与位置、工艺、材质、纹理、木纹、固有色、表面处理和 Logo；
- 目标角度会暴露哪些表面；
- 精确长宽高与关键局部尺寸是否已知；
- 允许变化项、未知项和禁止编造项。

硬门禁：

- 目标角度暴露输入未覆盖的侧面、背面、内部或底部结构时，只能标记 `concept_draft`；进入商业候选必须补照片或可靠 3D/CAD。
- 镜像复制、按常识补齐、用同类商品替代或凭风格猜测，不是产品依据。
- 缺精确尺寸可以做氛围校准候选，但不得标记 `approved_final`，不得添加尺寸线或暗示精确尺度。
- 产品关键细节无法辨认、输入互相冲突或用户要求修改锁定项时，状态为 `blocked`。

## 5. 不可变产品指纹

以下项目全部锁死：

- 整体造型、轮廓、长宽高关系、厚薄关系和各段比例；
- 部件数量、形状、尺寸关系、相对位置、开合与层级关系；
- 五金、拉手、脚架、铰链、滑轨、钉扣及其他可见配件；
- 榫接、拼接、包边、压条、收口和其他连接工艺；
- 缝线走向、针距观感、滚边、压边、倒角、边型和软包分缝；
- 材质类别、纹理尺度、纹理方向、木纹走向及跨部件连续关系；
- 固有色、透明度、光泽度、粗糙度、做旧、涂装及其他表面处理；
- Logo 的内容、大小、位置、朝向、颜色与清晰度。

光影只能改变受光明暗、高光和环境反射，不能借光影改变固有色、材质类别、纹理方向或表面处理。

## 6. 角度可变，但只能做刚体相机变换

产品必须被视为完整刚体。视角变化只能由整件产品旋转、平移、统一等比缩放和同一台相机投影产生。

严禁：

- 逐角贴合场景原家具；
- 非等比缩放、梯形拉伸、局部透视、液化、网格扭曲或单独移动角点；
- 分别调整门板、抽屉、台面、外框、底轨、立柱或脚架的倾斜角；
- 为制造“有角度”而强加左右高低差、板缝倾斜或明显梯形；
- 用高低脚、错层、局部拉长、阴影或道具掩盖几何错误。

刚性几何检查：

- 产品在三维物体空间中的平行结构仍然平行；
- 投影后同一方向的线只服从同一个消失点体系；
- 箱柜、桌几、床架等规则家具的外框、横竖分格、台面、底轨、拉手阵列和脚架保持原始直线、间距与矩形逻辑；
- 四脚、底座或其他承重点属于同一刚性结构，并落在同一可信地面；
- 近正面参考默认保持端正，只保留真实镜头必要的轻微透视；不复制参考图中的广角畸变、拍摄歪斜、滚动快门、后期拉正痕迹或原家具变形。

若参考机位与产品刚性几何无法同时成立，必须优先保持产品端正并降低角度拟合；仍会误导商品外观时停止生成。不得为了修复 P1 的角度不匹配制造 P0 的产品变形。

## 7. 五维分析

生成前和质检时都要完成以下五维判断，每一维写明证据、目标、风险与结论。

### 产品本体

检查造型、比例、刚性几何、部件、五金、连接、缝线边型、材质、纹理、木纹、固有色、表面处理、Logo、尺寸和目标视角覆盖。

### 场景搭配

检查空间功能、风格统一性、门窗墙地、动线、软装陈设、产品使用空间、透视、尺度和遮挡。北欧、中古、极简等风格必须自洽，道具不能暗示不存在的配套或功能。

### 用户与生活方式

判断谁在使用、什么生活动作、什么状态和情绪价值。人物、宠物与软装只能辅助生活方式表达，不能喧宾夺主或遮挡核心卖点。

### 卖点传达

明确核心卖点、第一视觉焦点、信息层级、遮挡风险和缩略图识别速度。关键结构、五金、材质和工艺不得被裁切、道具或阴影遮住。

### 商业定位

判断价格带暗示、品牌调性、目标渠道、画幅、裁切安全区和批次一致性。商业清晰度不能牺牲产品真实性。

## 8. 机位、透视与落地

场景分析必须记录：

- 地平线、消失点、垂直线、相机高度、俯仰、偏航、横滚和焦段感；
- 产品预计落位、可信尺度基准、地面方向与接触区域；
- 产品水平边和纵深边的消失关系；
- 前后景遮挡和景深关系。

执行规则：

- 产品尺度优先由精确尺寸和可信空间基准推导；不能通过拉伸或压扁把产品塞进空间。
- 所有承重点必须落在同一地面，具有正确接触、遮挡和短而稳定的环境遮蔽。
- 产品不得漂浮、下陷、穿模或被不合理遮挡。
- 角度迁移后先做灰度几何检查：外框是否直、顶底是否成套、分格是否齐、拉手阵列是否稳定、承重点是否共地。任一失败立即废稿，不继续美化。

## 9. 光影、材质与真实摄影

必须从场景参考提取：主光位置、方向、高度、面积、软硬、色温、强弱、衰减、反射光、阴影方向与边缘。

执行规则：

- 产品亮暗面、高光和投影服从同一光照因果；多个光源的每组影子都要能解释来源。
- 接触阴影集中在脚、底座和贴地面附近，短、稳且不脱离承重点。
- 环境反射只能造成轻微色彩和亮度变化，不能覆盖产品固有色。
- 木材保留木纹方向、尺度和表面处理；织物保留纤维与织法；皮革保留粒面、缝线和既有光泽；金属有符合环境的反射；玻璃保留透明、反射和厚度边缘；石材不得重造纹路。
- 产品与场景共享焦平面、景深、颗粒、锐度、反差和白平衡；不得出现抠图白边、光晕、双重阴影或无来源轮廓光。
- 最终必须像真实商业摄影：禁止 AI 塑料感、CGI 漂浮感、虚假 HDR、过度磨皮、过度降噪、全局假锐化、纹理重造、边缘糊化或靠浅景深隐藏错误。
- 原尺寸查看时，五金、拉手、连接件、接缝、边角、木纹/织纹、外框、底轨、脚架和表面质感仍须清楚可核对。

## 10. 生成流程

1. 自动区分产品图与场景参考图，并复述各自角色。
2. 建立产品锁定清单、表面覆盖、尺寸状态和未知项。
3. 完成五维分析，并记录机位、消失点、焦段感、光源与阴影。
4. 在生成指令中把不可变项写成可数、可定位、可对照的事实；不能只写“保持不变”。
5. 明确声明：场景原家具不是产品结构来源，产品只做整件刚体相机变换。
6. 先生成一张 `calibration_candidate`，不直接批量消耗额度。
7. 先检查 P0 刚性几何与产品身份，再检查 P1 物理融合，最后检查 P2 商业表达。
8. 人工明确确认校准规则后，才允许同一产品、同一视角范围、同一场景规则和同一用途批量扩展。

## 11. 状态门禁

只允许以下状态：

- `blocked`：无法安全继续；输入冲突、关键身份不清、要求修改锁定项或连续两次产品崩坏。
- `concept_draft`：只供方向讨论；存在未知面或关键依据不足，不能作为商品事实。
- `calibration_candidate`：第一张保真校准候选，尚未获得人工确认。
- `qa_passed_candidate`：人工已确认校准规则，且候选通过 P0/P1；仍不是最终成品。
- `approved_final`：精确尺寸和目标视角覆盖完整，P0/P1/P2 全部通过，并获得人工明确批准，可用于已声明用途。

未经人工确认，只能称“候选稿”，不得称“商业成品”“最终稿”或 `approved_final`。

## 12. P0 / P1 / P2

### P0 产品保真否决

任一项失败立即废稿：

- 轮廓、比例或刚性几何改变；
- 局部歪斜、弯折、梯形拉伸、非等比缩放或冲突消失点；
- 部件、五金、拉手、连接、缝线、边型、外框、底轨或脚架增删、错位、变形；
- 材质、纹理、木纹、固有色、表面处理或 Logo 错误；
- 关键细节被糊化、抹平、融合、重造或替换；
- 把未知面伪装成真实商品；
- 通过裁切、遮挡、阴影或景深掩盖产品错误。

### P1 商业可用性阻断

任一项失败不得进入商业候选：

- 未按参考场景的可解释机位与角度对齐；
- 尺度、透视、落地、遮挡、主光、投影或接触阴影冲突；
- 漂浮、下陷、穿模、抠图边、AI 塑料感或虚假锐化；
- 关键卖点被裁切；
- 出现未经证实的尺寸、文案、功能、认证、配套或价格；
- 画幅、色彩空间或交付规格错误。

### P2 优化项

轻微留白、道具节奏、局部色调和次要背景细节可列为 P2。P2 不能掩盖或降级 P0/P1；`approved_final` 前 P2 也必须处理完毕。

## 13. 质检与交付

每张候选按以下顺序检查，上一层不通过就停止：

1. 产品原图与结果图并排核对整体轮廓、比例和部件；
2. 原尺寸放大核对五金、拉手、连接件、接缝、边角、纹理、外框、底轨和脚架；
3. 核对未知面、精确尺寸和目标角度覆盖；
4. 核对刚性几何和统一相机投影；
5. 核对参考机位、空间物理、落地、光影和材质响应；
6. 核对生活方式、卖点、价格感、渠道、画幅与裁切安全区。

同一目标连续两次出现 P0 产品崩坏时，立即停止重试并标记 `blocked`；重新检查素材覆盖、目标视角或当前生成路径，不得无限消耗额度。

商品首图要快速识别产品轮廓和卖点；详情页场景图要解释真实使用关系；广告或社媒图可以增强氛围，但仍须通过全部保真门禁。不同画幅只能移动构图窗口或调整场景留白，不能对产品非等比缩放。

## 14. 每次对话的最小输出

生成前简短给出：

- 产品锁定项；
- 场景机位与光影结论；
- 未知项和当前状态。

生成后给出：

- 当前状态；
- P0 / P1 / P2 结论；
- 是否允许继续校准、批量或交付。

不得把未执行、未核对或未经人工确认的结果写成已经通过。
